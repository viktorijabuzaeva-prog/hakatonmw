# Инструкция по совместной работе с проектом

## Для владельца репозитория: Добавление участников

1. Откройте репозиторий: https://github.com/viktorijabuzaeva-prog/hakatonmw
2. Перейдите в **Settings** (шестерёнка)
3. В левом меню выберите **Collaborators**
4. Нажмите **Add people**
5. Введите GitHub username или email участника
6. Нажмите **Add to this repository**

Участник получит приглашение на email и сможет вносить изменения напрямую.

---

## Для участника: Начало работы

### 1. Установите Git
Скачайте и установите: https://git-scm.com/download/win

### 2. Настройте Git (один раз)
Откройте терминал (PowerShell или Git Bash) и выполните:

```bash
git config --global user.name "Ваше Имя"
git config --global user.email "ваш-email@example.com"
```

### 3. Склонируйте репозиторий
```bash
git clone https://github.com/viktorijabuzaeva-prog/hakatonmw.git
cd hakatonmw
```

### 4. Установите зависимости
```bash
cd backend
pip install -r requirements.txt
```

## Внесение изменений

### Шаг 1: Получите последние изменения
Перед началом работы всегда синхронизируйтесь:
```bash
git pull origin main
```

### Шаг 2: Внесите изменения
Редактируйте файлы в любом редакторе (VS Code, Cursor, PyCharm и др.)

### Шаг 3: Проверьте статус
```bash
git status
```

### Шаг 4: Добавьте изменения
```bash
# Добавить все изменённые файлы
git add .

# Или добавить конкретный файл
git add путь/к/файлу.py
```

### Шаг 5: Создайте коммит
```bash
git commit -m "Описание изменений"
```

Примеры хороших сообщений:
- `"Добавлен анализ новых транскриптов"`
- `"Исправлена ошибка в парсере"`
- `"Обновлён интерфейс фильтрации"`

### Шаг 6: Отправьте на GitHub
```bash
git push origin main
```

При первом push Git попросит авторизоваться — введите логин и пароль GitHub.

---

## Решение конфликтов

Если кто-то уже внёс изменения в те же файлы:

```bash
# Получите изменения
git pull origin main

# Git покажет конфликтные файлы
# Откройте их и найдите маркеры:
# <<<<<<< HEAD
# ваши изменения
# =======
# чужие изменения
# >>>>>>> origin/main

# Отредактируйте файл, оставив нужный код
# Удалите маркеры конфликта

# Затем:
git add .
git commit -m "Разрешён конфликт в файле X"
git push origin main
```

## Полезные команды

| Команда | Описание |
|---------|----------|
| `git status` | Показать изменённые файлы |
| `git log --oneline` | История коммитов |
| `git diff` | Показать изменения в файлах |
| `git pull` | Получить изменения с GitHub |
| `git push` | Отправить изменения на GitHub |
| `git checkout -- файл` | Отменить изменения в файле |

## Работа с транскриптами

⚠️ **Важно:** Папка `Transcripts/` добавлена в `.gitignore` и не загружается на GitHub.

Каждый участник должен:
1. Создать папку `Transcripts/` локально
2. Добавить туда .docx файлы с интервью
3. Эти файлы останутся только на локальном компьютере
